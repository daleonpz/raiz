# Example

Example tree

```bash
$ tree -L 2
  .
 ├──  Makefile
 ├──  math.c
 ├──  math.h
 ├── 󰂺 README.md
 ├──  requirements
 │   └──  requirements.yaml
 └──  robot
     ├──  __pycache__
     ├──  MathLibrary.py
     ├──  test_math.robot
     └──  test_multiplication.robot
```

* `Makefile`* is used to build example and generate robot results.
* `math.c` and `math.h` are the source files of the example.
* `requirements/requirements.yaml` is the requirements file. When using raiz for first time, it will be created.
* `robot/` contains the robot tests and the library used by them.

```
cd example
make
make robot
```

Some test are going to fail, that's ok, this is just an example of how to use the raiz.

Let's load requirements.yaml to raiz

```bash
$ raiz sync from-yaml
```

Now there are new files and folders
```bash
$ tree -L 2

├── 󰗀 output.xml

├──  requirements
│   └──  requirements.yaml
└──  robot
    ├──  __pycache__
    ├──  MathLibrary.py
    ├──  test_math.robot
    └──  test_multiplication.robot
```

* `.req_cache/` is the cache of raiz, it contains the requirements database.
* `build/` is the folder where the compiled library is stored.
* `log.html`, `output.xml`, and `report.html` are the logs and reports generated by robot framework.
